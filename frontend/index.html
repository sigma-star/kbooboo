Notes on submitting bug reports:

  • If you run a Linux kernel provided by your Linux distribution, it's best to report the issue first to their bug tracker.
  • Read Linux'sReporting issues documentation. You can skip reading the MAINTAINERS file manually since this web based tool can extract the maintainer entries from kernel crash logs.
  • Especially read the section on decoding stack traces.
  • Send Email in plain text. See https://useplaintext.email for more information.
  • Always CC linux-kernel@vger.kernel.org.
  • Don't skip the mailing lists, writing privately to kernel maintainers is considered rude.
  • Some subsystems provide a chat contact, mostly IRC. If you want to read more on IRC, see New to Using IRC.
  • If you have further questions, feel free to join #kernelnewbies on the irc.oftc.net IRC network.
  • Be patient.
